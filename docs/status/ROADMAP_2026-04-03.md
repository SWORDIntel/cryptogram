# CRYPTOGRAM Roadmap — Updated 2026-06-27

## Current State: Production-Ready Build & Release Pipeline

As of June 2026, CRYPTOGRAM has achieved the following milestones:

- **Desktop (Linux)**: Builds successfully with Qt6, OpenSSL 3.x, CMake/Ninja. All counterintelligence, security, and protocol modules are compiled and linked.
- **Android**: Builds with Gradle + CMake (NDK 25.2). Native `libcryptogram.so` includes real Double Ratchet (X3DH + DH ratcheting + chain key ratcheting + AES-256-GCM) and MLS (TreeKEM + group encryption).
- **CI/CD**: Two production GitHub Actions workflows (`linux-deb.yml`, `android-apk.yml`) produce signed `.deb` and signed APK artifacts, attached to GitHub Releases on tag pushes.
- **Testing**: 205 verification checks pass across `run_tests.sh` (80 static checks) and `run_e2e_tests.sh` (125 E2E checks). Six Catch2 test files cover crypto primitives, Signal Protocol, MLS Protocol, build verification, Android JNI surface, and counterintelligence framework.
- **Signing**: Android release APKs are signed via GitHub repository secrets (4096-bit RSA keystore, 30-year validity).
- **Documentation**: README, CHANGELOG, and docs are aligned with repo reality. All source files are wired into CMakeLists.txt.
